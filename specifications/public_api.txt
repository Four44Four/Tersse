 - Only expose UI element creation, deletion, property modification (text display, color, position, size), focus number changing, focused element changing, lock status, etc...
 - Do not expose any internal constants, keyboard processing/input logic, ID allocation logic, and other mechanisms that are not directly relevant to UI flow
    - Keep clipboard API exposed
 - Expose a function to queue a another function/closure to run synchronously in the context of the UI thread
    - The running of that inputted function/closure will trigger a UI update immediately after it concludes running
    - This API function should be safely callable from any thread or async environment to trigger a UI update asynchrously or concurrently without causing race conditions within UI
 - Expose a function to send a message string
    - This will trigger the messsage gutter to either appear, or be updated
    - See ./specifications/message_gutter.txt
    - Should be thread-safe
       - Can call from async or multi-threaded environments
 - Expose terminal_size() for reading the current terminal dimensions
 - Expose register_terminal_resize_callback on RuntimeUi to register callbacks that run on the UI thread after a debounced terminal resize when dimensions change (not on initial startup sync)
